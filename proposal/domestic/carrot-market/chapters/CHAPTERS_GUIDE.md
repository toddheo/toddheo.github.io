# 챕터 작성 가이드

각 챕터 파일은 `/new-proposal` 스킬 실행 시 자동 생성됩니다.
아래는 각 챕터의 필수 구성 요소 가이드입니다.

---

## 공통 규칙

- 각 파일 최상단: `**[YYYY-MM-DD]**`
- 금지 단어: 금융감독원(금감원), 규제 샌드박스
- 스테이지: **5·7·9·11·13개월 홀수 순번계만** 사용
- 납입 범위: **월 10만 원 ~ 최대 160만 원**
- 수치 기준일: **2026년 4월**

## 챕터별 핵심 구성

### 01_background.md
- 거시 환경 변화 (정책·사회·경제 트렌드)
- 아임인 도입 필요성 도출

### 02_problem.md
- 핵심 페인포인트 3가지 (데이터·수치 기반)
- 문제의 심각성과 파급력

### 03_current_limitations.md
- 현행 제도/서비스의 3가지 구조적 한계
- 경쟁 대안 비교표

### 04_urgency.md
- 골든타임 논거 (지금 도입 안 하면?)
- 손실 규모 추정

### 05_objectives.md
- 도입 목적 (3~5가지)
- 아임인의 전략적 포지셔닝

### 06_core_values.md
- 3축: 안전성 / 유동성 / 소셜 동기부여
- 각 축별 근거와 군/기관 적용 의미

### 07_service_overview.md
- 아임인 서비스 개요
- 스테이지 운용 원리 (9명 그룹 예시)
- 주요 기능 상세

### 08_scenarios.md
- 대상 유저 유형별 시나리오 3~4가지
- 각 시나리오: 납입액·스테이지·수령액 계산

### 09_safety.md
- KCB 신용평가 연동
- 안심서비스 구조
- 0.22% 연체율 비교

### 10_cooperation.md
- 기관 R&R / 티웨이브 R&R 분담표
- 운영 거버넌스

### 11_roadmap.md
- 준비(3개월) → 파일럿(6개월) → 확대(6개월) → 정착
- 단계별 마일스톤

### 12_pilot.md
- 파일럿 대상 선정 기준
- 운영 프로세스
- KPI 측정 방법

### 13_expansion.md
- 확대 기준 및 조건
- 전체 도입 시나리오

### 14_quantitative.md
- 참여 규모별 효과 시뮬레이션 표
- ROI 분석

### 15_qualitative.md
- 사기 진작 / 문화 변화 / 사회적 효과
- 실사용자 후기 인용

### 16_cases.md
- 실제 아임인 사용자 성공 사례 3~5건
- 대상 유저 적용 시나리오로 재해석

### 17_company.md
- 기업 기본 정보 표
- 연혁 (2016~2026년)

### 18_achievements.md
- 핵심 성과 지표 박스 (ASCII 표)
- 거래 규모 성장 추이 표
- 연체율 비교 표

### 19_partnerships.md
- 신한은행, 케이뱅크 파트너십
- 수상 이력 (신한퓨처스랩 10기 등)
- 언론 보도

### appendix/A_legal.md
- 전자금융거래법, 개인정보보호법, 신용정보법 준수 현황
- 기관별 추가 규정 검토

### appendix/B_simulation.md
- 1인당 저축 시뮬레이션 (이병→병장 or 동등 대상)
- 전체 도입 시 연간 효과
- ROI 분석
