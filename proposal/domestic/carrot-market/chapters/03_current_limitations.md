# 1.3 현행 대안의 구조적 한계

---

## 현재 당근 유저가 쓰는 대안들

동네 이웃과 함께 자금을 모으거나 목돈을 마련하려는 당근 유저들이 현재 사용하는 서비스와 그 한계를 분석합니다.

| 현행 대안 | 표면적 기능 | 구조적 한계 |
|----------|------------|------------|
| **카카오페이 모임통장** | 그룹 공동 계좌 | 계모임 구조 없음, 잔액 공유에 그침. 당근 생태계 이탈 |
| **카카오뱅크 모임통장** | 목적별 저금통 | 개인 저축 중심, 순번 지급 구조 없음, 공동 납입 불가 |
| **토스 저금통** | 자동이체 저축 | 1인 상품, 소그룹 동기부여 없음 |
| **개인 은행 적금** | 이자 수익 | 목적 자금 마련 타이밍 불일치, 중도 해지율 높음 |
| **개인 계좌 취합** | 비용 0원 | 분쟁 리스크, 투명성 부재, 관리 부담 |
| **당근페이 현재 기능** | 결제·송금 | 공동 저축·ROSCA 구조 없음 |

---

## 핵심 공백: 소셜 동기부여 + 목적 저축의 결합

기존 서비스는 두 가지 중 하나만 제공합니다.

```
         [소셜 동기부여]
              ↑
              │
     아임인   ●  ← (이 영역이 비어 있음)
              │
──────────────┼────────────────→ [목적 저축]
              │
   개인 적금  ●    모임통장 ●
              │
              ↓
         [개인 단독]
```

- **개인 적금·토스**: 목적 저축은 있지만 소셜 동기부여 없음 → 중도 포기율 높음
- **카카오 모임통장**: 소셜 요소는 있지만 ROSCA 구조(순번 지급)가 없어 동기 지속이 어려움
- **아임인**: 소셜 동기부여 + 목적 저축 + 순번 지급이라는 **3축이 동시에 작동**

## 당근 생태계 이탈 비용

현재 당근 유저가 모임 자금을 카카오페이·토스로 관리할 때 발생하는 기회 손실:

| 손실 유형 | 내용 |
|----------|------|
| **체류 시간 이탈** | 자금 관리를 위해 당근 앱을 나가는 행동 반복 |
| **GMV 손실** | 모임 수령금이 타 플랫폼 결제로 소비됨 |
| **브랜드 연결 단절** | 모임 경험이 당근이 아닌 카카오·토스와 연결됨 |
| **데이터 손실** | 모임 자금 흐름 데이터가 당근 생태계 외부에 축적됨 |

> 아임인과의 파트너십은 이 이탈을 당근 생태계 내부로 **되돌리는** 전략입니다.
