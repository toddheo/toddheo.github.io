# 2.6 협업 구조 (R&R)

---

## 3단계 파트너십 구조

### 1단계: 바터 트래픽 교환 (즉시 실행)

| 역할 | 내용 |
|------|------|
| **당근마켓** | 당근 앱(동네생활·당근페이 탭) 내 아임인 제휴 배너 노출 |
| **티웨이브** | 아임인 플랫폼 내 당근마켓 제휴 배너 노출 |
| **혜택 구조** | 당근→아임인 전환: 전용 스테이지 쿠폰 5만 원 (티웨이브 부담) / 아임인→당근 전환: 당근머니 쿠폰 (당근 제공) |
| **수익 모델** | 상호 무상 제공, CPA 전환 기반 |
| **실행 속도** | 결정 즉시 **2주 이내** 집행 가능 (시스템 연동 불필요) |

---

### 2단계: 당근페이 × 아임인 수령금 연동

| 역할 | 내용 |
|------|------|
| **당근마켓** | 당근페이 SDK·API 연동 창구 제공, 당근머니 전환 기능 개발 협조 |
| **티웨이브** | 아임인 수령금 → 당근머니 전환 옵션 개발, 운영 |
| **혜택 구조** | 수령금을 당근머니로 전환 시 **3~5% 추가 보너스** (협의) |
| **수익 모델** | 당근: GMV 증가 + 당근페이 거래량 상승 / 티웨이브: 신규 AU 확보 |
| **준비 기간** | API 연동 개발 약 **1~2개월** |

---

### 3단계: 동네생활 모임통장 화이트라벨링 (장기)

| 역할 | 내용 |
|------|------|
| **당근마켓** | 동네생활 탭 내 모임 자금 지면 제공, 당근 브랜드 적용 |
| **티웨이브** | 아임인 ROSCA 시스템 화이트라벨링, 운영·CS 전담 |
| **브랜딩** | '당근 모임통장' 등 당근 자체 브랜드 운영 ('Powered by imin' 협의) |
| **수익 모델** | 당근: 비용 0원, 동네생활 DAU 증가 / 티웨이브: 수수료 수취, AU 확보 |
| **준비 기간** | 법무 검토 + 개발 약 **3~6개월** |
