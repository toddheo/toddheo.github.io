# 4.1 기관 간 협력 체계 및 역할 분담

**[2026-05-08]**

---

## 4.1.1 협력 체계 전체 구조도

아임인 군 도입을 위한 협력 체계는 **국방부를 총괄 주체로**, 군 관련 기관과 (주)티웨이브가 명확한 역할을 분담하는 구조로 설계됩니다.

```
┌────────────────────────────────────────────────────────────────────┐
│                     협력 체계 전체 구조                               │
│                                                                       │
│   ┌─────────────┐          ┌─────────────────────────────────┐      │
│   │  국방부      │          │          (주)티웨이브            │      │
│   │ (정책 총괄)  │◄────────►│     (플랫폼 운영 총괄)           │      │
│   └──────┬──────┘          └────────────────┬────────────────┘      │
│          │                                   │                        │
│    ┌─────┴─────────────────┐                 │                        │
│    │                       │                 │                        │
│    ▼                       ▼                 │                        │
│ ┌──────────┐         ┌──────────┐            │                        │
│ │ 각 군    │         │ 국군     │            │                        │
│ │ (육·해·공│         │ 복지단  │            │                        │
│ │ ·해병대) │         │         │            │                        │
│ └──────┬───┘         └────┬─────┘            │                        │
│        │                  │                  │                        │
│        ▼                  ▼                  ▼                        │
│   ┌──────────────────────────────────────────────┐                   │
│   │               장병 (최종 수혜자)               │                   │
│   └──────────────────────────────────────────────┘                   │
└────────────────────────────────────────────────────────────────────┘
```

## 4.1.2 기관별 역할 분담 상세

### ① 국방부 (정책 총괄 기관)

| 역할 구분 | 세부 내용 |
| --------- | --------- |
| **정책 결정 및 승인** | 아임인 군 도입 정책 결정, 시범 운영 부대 지정 |
| **제도적 기반 마련** | 군내 소셜 저축 플랫폼 이용 관련 내규 정비 |
| **성과 평가 총괄** | 파일럿 성과 지표 설정, 전군 확대 여부 최종 판단 |
| **대외 홍보** | 장병 복지 개선 정책의 일환으로 공식 홍보 |
| **예산 배정** | 금융 교육 프로그램 운영 예산 확보 (필요 시) |

---

### ② 각 군 (육·해·공군·해병대)

| 역할 구분 | 세부 내용 |
| --------- | --------- |
| **부대별 시범 운영 주관** | 해당 군 내 파일럿 부대 선정 및 운영 관리 |
| **장병 홍보 및 교육** | 각 부대 내 아임인 서비스 소개 및 가입 유도 |
| **의견 수렴 채널 운영** | 장병의 서비스 이용 만족도 및 개선 의견 취합 |
| **보안 검토** | 군 보안 규정 준수 여부 검토 및 승인 |

---

### ③ 국군 복지단

| 역할 구분 | 세부 내용 |
| --------- | --------- |
| **장병 복지 서비스 연계** | 기존 복지 서비스와 아임인 연계 방안 설계 |
| **금융 교육 프로그램 공동 개발** | (주)티웨이브와 협력하여 군 맞춤형 금융 교육 콘텐츠 제작 |
| **이용 지원 채널 운영** | 아임인 이용 관련 장병 문의 1차 지원 |
| **특화 혜택 협의** | 군 장병 전용 가입 혜택, 이벤트 기획 협의 |

---

### ④ (주)티웨이브 (서비스 제공사)

| 역할 구분 | 세부 내용 |
| --------- | --------- |
| **플랫폼 운영 총괄** | 아임인 서비스 안정적 운영 및 기술 지원 |
| **군 특화 기능 개발** | 필요 시 군 장병 전용 UI/UX 및 기능 개발 |
| **금융 교육 콘텐츠 제공** | 군 맞춤형 재무 교육 콘텐츠 설계 및 제공 |
| **성과 데이터 제공** | 파일럿 성과 지표 데이터 정기 보고 |
| **고객 지원** | 장병 이용자 직접 지원 채널 운영 |
| **컴플라이언스 관리** | 개인정보 보호 및 금융 법규 준수 총괄 |

## 4.1.3 정보 흐름 및 보고 체계

```
[장병 이용 데이터] → (주)티웨이브 → 성과 보고서 → 국군복지단 → 국방부
         │
         ▼
[만족도·개선 의견] → 각 군 의견 수렴 → 서비스 개선 반영 → (주)티웨이브
```

### 정기 보고 주기

| 보고 유형 | 주기 | 보고 기관 | 수신 기관 |
| --------- | :--: | --------- | --------- |
| 이용자 현황 보고 | 월 1회 | (주)티웨이브 | 국군복지단 |
| 만족도 조사 결과 | 분기 1회 | 각 군 | 국방부 |
| 종합 성과 평가 | 반기 1회 | 국군복지단 | 국방부 |
| 전군 확대 검토 | 연 1회 | 국방부 주관 | 최고 정책 결정 |

## 4.1.4 업무협약(MOU) 체결 주요 항목 (안)

국방부 및 관련 기관과의 업무협약 시 포함될 주요 항목을 사전에 제안합니다.

| MOU 항목 | 내용 |
| --------- | ---- |
| **협력 목적** | 군 장병의 건전한 재무 습관 형성 및 금융 사고 예방 |
| **협력 범위** | 아임인 서비스 군 장병 이용 지원, 금융 교육 공동 시행 |
| **개인정보 보호** | 장병 개인정보의 엄격한 분리 관리, 군사 정보와 완전 분리 |
| **독점성 조항** | 시범 운영 기간 중 유사 서비스 중복 도입 유예 |
| **성과 공유** | 파일럿 데이터 및 성과 지표의 상호 공유 |
| **계약 기간** | 최초 1년, 성과 평가 후 자동 연장 가능 |
| **비용 부담** | 서비스 운영 비용: (주)티웨이브 자체 부담 / 금융 교육 예산: 협의 |

## 4.1.5 군 보안 관련 우려사항 사전 해소

국방부가 가질 수 있는 보안 관련 우려사항과 이에 대한 사전 해소 방안을 제시합니다.

| 우려사항 | 해소 방안 |
| -------- | --------- |
| **장병 개인정보 유출** | 이름·연락처 외 군 관련 정보(부대, 계급, 복무 기간 등) 미수집 |
| **군사 기밀 연결 가능성** | 서비스 내 군사 관련 정보 전혀 불포함, 일반 민간 금융 앱과 동일 구조 |
| **스마트폰 보안 정책 충돌** | 기존 스마트폰 허용 부대의 앱 화이트리스트에 등록 신청 가능 |
| **외부 서버 데이터 저장** | 국내 IDC 서버에만 저장, 국외 전송 없음 |
| **해킹 및 사이버 공격** | ISO 27001 보안 인증 수준의 인프라 운영, 정기 보안 감사 수행 |
