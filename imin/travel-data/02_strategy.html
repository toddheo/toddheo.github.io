1. 개요 및 배경

추진 배경

아임인 사용자는 스테이지 참여 시 목적(여행, 결혼, 육아 등)을 설정합니다. '여행' 목적으로 스테이지에 참여한 사용자는 구조적으로 다음을 확정적으로 알 수 있습니다:

알 수 있는 정보구체적 내용여행업체 활용 가치
수령 시점스테이지 회차 구조상 특정 연·월에 목돈 수령 확정프로모션 타이밍 최적화
수령 금액스테이지 납입 구조로 수령 금액 사전 확정적합 상품 레코멘데이션
여행 인텐트사용자가 직접 '여행' 목적으로 저축 중전환율 극대화

추진 목적

① 아임인의 데이터 자산을 B2B 수익원으로 전환 — 데이터 라이선스 수수료 창출
② 사용자에게 목돈 수령 시점의 여행 혜택을 제공하여 플랫폼 가치 향상
③ 마이리얼트립·트립닷컴·클룩 등과의 전략적 파트너십 구축 → 향후 화이트라벨·공동 캠페인 확장 기반

2. 타겟 세그먼트 분석

항목내용
정의스테이지 목적을 '여행'으로 설정하고 현재 납입 중인 활성 사용자
추정 규모활성 사용자 약 60만 명 × 여행 목적 비중 ~15% = 약 9만 명
수령 예정 금액 분포100만원대 40% / 200만원대 35% / 300만원 이상 25%
수령 예정 시점 분포3개월 이내 25% / 3~6개월 30% / 6~12개월 45%
구매력 특성이미 여행 예산을 저축 완료한 확정적 여행 소비자
경쟁 우위일반 여행 앱의 '관심 사용자'와 달리 예산 확보가 확정된 고인텐트 세그먼트

5. 리스크 분석

🔴 개인정보·법규 리스크
  • 개인정보보호법 제17조 제3자 제공 동의 위반 시 형사처벌 가능
  • 동의 없는 데이터 제공 = 과징금 (매출의 3% 이내)
  • 금융데이터로 오해될 경우 신용정보법 저촉 가능성
🔴 사용자 신뢰 리스크
  • 동의 과정 불투명 시 "아임인이 내 정보 팔았다" 인식 확산
  • SNS 역풍 시 기존 사용자 이탈 가능성
  • 동의율 저조 시 파트너 기대치 미달 → 계약 파기
🟠 데이터 품질 리스크
  • 스테이지 중도 탈퇴자 → 수령 시점 변경으로 파트너사 데이터 신뢰도 저하
  • 여행 목적 스테이지 분류 정확도 의존성
  • 연락처 오류·미입력자 제외 시 실 제공 규모 축소
🟠 파트너십 리스크
  • 여행업체가 데이터 가치를 낮게 평가 시 단가 협상 실패
  • 파트너의 무분별한 마케팅으로 사용자 불만 유발
  • 파트너 간 데이터 재판매·유출 리스크

6. 실행 계획

Phase 1 — 준비 (2026년 6~7월)

법률 검토 및 내부 인프라 구축

  • 개인정보보호법 법무 검토 완료 및 동의서 최종 확정
  • CPO: 여행 목적 스테이지 태그 정확도 검증 및 데이터 파이프라인 설계
  • 동의 수집 UI 개발 (앱 내 팝업 + 마이페이지 설정)
  • 마이리얼트립 1개사 대상 파일럿 파트너십 MOU 체결
Phase 2 — 파일럿 (2026년 8~10월, 3개월)

마이리얼트립 1사 대상 파일럿 운영

  • 여행 목적 스테이지 사용자 대상 인앱 동의 수집 캠페인
  • 동의자 데이터 → 마이리얼트립에 월 1회 배치 제공
  • 마이리얼트립 → 아임인 동의자 대상 여행 상품 발송 (월 2회 한도)
  • KPI 측정: 동의율, 오픈율, 전환율, 사용자 불만 건수
Phase 3 — 확장 (2026년 11월~)

트립닷컴·클룩 추가 계약 및 상품 고도화

  • 파일럿 성과 기반 단가 협상 및 계약 확대
  • 실시간 API 제공 방식으로 전환 (배치 → API)
  • CPA 전환 수수료 모델 추가 도입
  • 보험·렌터카·환전 등 인접 산업으로 파트너 확장
